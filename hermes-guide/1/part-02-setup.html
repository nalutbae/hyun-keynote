Provider별 특징

항목확인할 것주의할 점
OpenAIAPI key, 모델명, 응답 품질rate limit와 과금 기준을 함께 본다

Docker 필요성

초보자에게 중요한 질문은 "Docker를 쓸 수 있나"가 아니라 "왜 Docker가 필요한가"입니다.

서버에서 계속 띄우거나, gateway를 background로 오래 운영하거나, 설정/세션/skill/memory를 한 디렉터리에 모으거나, 위험한 명령 실행을 분리해야 할 때 Docker가 유용합니다.

검증의 중요성

검증은 "프로세스가 살아 있다"에서 끝나면 안 됩니다. gateway가 살아 있어도 Slack thread로 답이 돌아오지 않을 수 있고, cron job이 등록되어 있어도 fresh session에서 실패할 수 있습니다.

CLI → Provider → Toolset → Gateway → Cron 순서로 한 층씩 검증하는 것이 안전합니다.

마무리

Hermes Agent 운영은 기능을 한 번에 켜는 일이 아니라, 작동 범위를 한 층씩 넓히는 일입니다.

CLI가 불안정한데 Gateway를 붙이면 문제가 메시징인지 provider인지 모델인지 알기 어렵습니다. Gateway가 불안정한데 Cron을 붙이면 예약 실행 문제인지 delivery 문제인지 헷갈립니다.

한 층씩, 안정되면 다음 층을. 이것이 Hermes Agent 운영의 핵심 원칙입니다.
