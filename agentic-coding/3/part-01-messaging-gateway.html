"게이트웨이는 하나의 프로그램입니다. 텔레그램에서도, 슬랙에서도, 디스코드에서도 같은 AI 에이전트와 대화할 수 있는 이유가 바로 이 게이트웨이 덕분입니다."

"음성 메시지 처리도 중요합니다. 텔레그램에서 음성 메시지를 보내면, 게이트웨이가 이를 텍스트로 변환해서 AI가 이해할 수 있게 만듭니다."

게이트웨이 아키텍처

01

메시지 수신
→ 플랫폼 어댑터

각 플랫폼(Telegram, Slack, Discord 등)에서 들어오는 메시지를 플랫폼 어댑터가 표준 형식으로 변환합니다. 플랫폼마다 API, 데이터 형식이 다르지만 어댑터가 통일합니다.

02

세션 스토어
→ AIAgent

변환된 메시지는 세션 스토어에서 대화 컨텍스트와 결합되어 AIAgent로 전달됩니다. 사용자별, 채널별 대화 기록이 유지됩니다.

03

크론 스케줄러

60초마다 크론 작업을 확인합니다. 정해진 시간에 메시지를 발송하거나, 주기적인 데이터 수집, 알림 등의 예약 작업을 수행합니다.

WeCom / Weixin

채팅 명령어 18개

/new

새 세션 시작

현재 대화 컨텍스트를 초기화하고 새로운 대화를 시작합니다.

/model

모델 전환

사용할 AI 모델을 변경합니다. GPT-4o, Claude, Gemini 등 전환 가능.

/retry

재시도

마지막 응답을 다시 생성합니다. 응답이 만족스럽지 않을 때 사용.

/undo

실행 취소

마지막 요청과 응답을 대화 기록에서 제거합니다.

/status

상태 확인

현재 세션 상태, 사용 모델, 토큰 사용량 등을 표시합니다.

/stop

응답 중지

AI가 응답 생성 중일 때 즉시 중지합니다.

/approve

승인

AI가 요청한 도구 실행을 승인합니다. 자동 승인이 꺼져 있을 때 사용.

/deny

거부

AI가 요청한 도구 실행을 거부합니다.

/sethome

홈 디렉토리

AI가 기본으로 작업하는 디렉토리를 설정합니다.

/compress

컨텍스트 압축

대화 기록이 길어지면 중요한 내용만 남기고 압축합니다.

/title

세션 제목

현재 대화 세션의 제목을 설정하거나 변경합니다.

/resume

세션 이어하기

이전에 종료된 세션을 이어서 대화를 계속합니다.

/usage

사용량 확인

토큰 사용량, API 호출 횟수 등 사용 통계를 표시합니다.

/insights

인사이트

AI가 분석한 대화 인사이트와 요약을 제공합니다.

/reasoning

추론 과정

AI의 추론 과정을 표시합니다. 어떻게 답을 도출했는지 확인.

/voice

음성 모드

음성 입력/출력 모드를 전환합니다. 오디오 대화가 가능.

/rollback

롤백

여러 턴 전의 대화로 되돌아갑니다. /undo는 1턴만 취소.

/background

백그라운드

긴 작업을 백그라운드로 보냅니다. 완료 시 알림을 받습니다.

세션 관리와 리셋 정책

01

세션 지속성

게이트웨이는 사용자별·채널별로 대화 컨텍스트를 유지합니다. 앱을 닫았다 열어도 이전 대화를 이어서 진행할 수 있습니다. /resume 명령어로 세션을 복원합니다.

02

리셋 정책

Daily 4AM: 매일 새벽 4시에 자동 리셋. 하루의 대화를 초기화합니다.
Idle 1440분: 24시간(1440분) 동안 대화가 없으면 자동 리셋.
Both: 두 조건 중 먼저 도달하는 것으로 리셋. 기본값입니다.

03

플랫폼별 오버라이드

각 플랫폼마다 다른 리셋 정책을 적용할 수 있습니다. 예를 들어 텔레그램은 Daily만, 슬랙은 Idle만 설정하는 것이 가능합니다.

04

/new와 /undo

/new는 즉시 새 세션을 시작하고, /undo는 마지막 턴만 취소합니다. 긴 대화에서 /compress를 사용하면 컨텍스트를 압축하여 토큰을 절약할 수 있습니다.

텔레그램: Polling vs Webhook

Polling 모드

클라이언트가 주기적으로 확인

  • 방향: 게이트웨이 → 텔레그램 API
  • 동작: 60초마다 텔레그램에 "새 메시지 있어?"라고 물어봄
  • 용도: 개발/테스트 환경, 공개 IP가 없는 환경
  • 유휴 비용: 메시지가 없어도 요청이 발생
  • 설정: 공개 URL 불필요, 방화벽 설정 없음
Webhook 모드

서버가 이벤트를 밀어줌

  • 방향: 텔레그램 → 게이트웨이
  • 동작: 새 메시지가 도착하면 텔레그램이 게이트웨이로 전송
  • 용도: 운영 환경, 낮은 지연이 필요한 경우
  • 유휴 비용: 메시지가 없으면 요청 없음
  • 설정: HTTPS 공개 URL 필요, SSL 인증서 필수
