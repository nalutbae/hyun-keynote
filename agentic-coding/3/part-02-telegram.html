Step 1: BotFather에서 봇 생성

01

BotFather 접속

텔레그램에서 @BotFather를 검색하고 대화를 시작합니다. 텔레그램 공식 봇 관리 도구입니다.

02

/newbot 명령어

/newbot을 입력하여 새 봇 생성을 시작합니다. BotFather이 단계별로 안내합니다.

03

표시 이름 선택

봇의 표시 이름(display name)을 입력합니다. 예: "Hermes Agent". 한글, 영문, 이모지 모두 가능합니다.

04

사용자 이름 & 토큰

사용자 이름(username)은 bot으로 끝나야 합니다. 예: hermes_my_bot. 생성 완료 후 API 토큰이 발급됩니다.

⚠️ 토큰 형식: 123456789:ABCdef... — 절대 공개 저장소에 올리지 마세요!

Step 2: 봇 커스터마이징

01

/setdescription

봇의 설명을 설정합니다. 봇 정보 페이지에 표시되는 상세 설명입니다.

02

/setabouttext

봇의 간단한 소개를 설정합니다. "이 봇은 무엇을 하나요?"에 답하는 짧은 문장입니다.

03

/setuserpic

봇의 명령어 목록을 설정합니다. 사용자가 "/"를 입력하면 자동완성 목록에 표시됩니다.

05

/setprivacy

Privacy Mode를 설정합니다. 그룹에서 봇이 모든 메시지를 수신할지 선택합니다.

✦

추천 명령어 목록

help, new, sethome, model, stop, status, retry, undo

온라인/오프라인 상태 표시기

01

설정 방법

YAML 설정 파일에 다음을 추가합니다:

status_indicator: true
status_online: "🟢 Hermes 온라인"
status_offline: "🔴 Hermes 오프라인"

게이트웨이가 시작/종료될 때 자동으로 상태 메시지를 변경합니다.

02

제한사항

전역 프로필 — 상태는 모든 채팅에 동일하게 표시됩니다. 채팅별로 다른 상태를 설정할 수 없습니다.

정상 종료 시만 — 게이트웨이가 정상적으로 종료될 때만 오프라인으로 변경됩니다. 강제 종료(Ctrl+C가 아닌 kill -9 등) 시에는 오프라인 메시지가 전송되지 않을 수 있습니다.

커맨드 메뉴 관리

01

max_commands

텔레그램이 한 번에 표시할 수 있는 명령어 수의 최대값입니다.

기본값: 60

Hermes가 등록하는 명령어가 많을 경우 이 값을 조정합니다. 텔레그램 API 제한으로 인해 너무 많은 명령어는 등록할 수 없습니다.

02

priority_mode

Hermes 명령어와 사용자 정의 명령어의 우선순위를 설정합니다.

prepend — Hermes 명령어를 맨 앞에 배치
append — Hermes 명령어를 맨 뒤에 배치
replace — Hermes 명령어만 표시

03

priority 리스트

priority 설정으로 특정 명령어의 순서를 직접 지정할 수도 있습니다.

예: priority: [new, model, help, stop]

자주 사용하는 명령어를 상단에 배치하면 접근성이 향상됩니다.

Webhook 모드

Polling (기본값)
  • 게이트웨이 → 텔레그램 방향으로 아웃바운드
  • 게이트웨이가 주기적으로 텔레그램 서버에 "새 메시지 있나요?"라고 물어봄
  • 항상 실행 상태여야 함
  • 설정이 간단, 추가 URL 불필요
  • 로컬 개발에 적합
Webhook
  • 텔레그램 → 게이트웨이 방향으로 인바운드
  • 새 메시지가 있으면 텔레그램이 게이트웨이로 푸시
  • 유휴 시간에는 절전 가능
  • 공개 HTTPS URL 필요
  • 클라우드 배포에 적합

Webhook 설정

01

TELEGRAM_WEBHOOK_URL

텔레그램이 메시지를 푸시할 HTTPS URL입니다.

예: https://your-domain.com/webhook/telegram

유효한 SSL 인증서가 필요합니다. Let's Encrypt를 추천합니다.

02

TELEGRAM_WEBHOOK_SECRET ← 필수

웹훅 요청을 검증하는 비밀 토큰입니다.

임의의 문자열을 설정합니다. 예: my-secret-token-123

홈 채널 설정

01

/sethome 명령어

채팅에서 직접 /sethome을 입력하면 현재 채팅이 홈 채널로 설정됩니다.

가장 간단한 방법이며, 즉시 적용됩니다.

홈 채널은 크론 작업의 결과가 전송되는 기본 채널입니다.

02

수동 설정

환경변수로 직접 설정할 수도 있습니다.

TELEGRAM_HOME_CHANNEL=-1001234567890

채널 ID는 숫자 형식이며, 슈퍼그룹은 -100 접두사가 붙습니다.

채널 ID는 @userinfobot이나 텔레그램 API로 확인할 수 있습니다.

03

크론 작업 연동

홈 채널은 크론 스케줄러의 결과 전송 채널입니다.

"매일 오전 9시 날씨 알림" 같은 크론 작업의 결과가 홈 채널로 전송됩니다.

홈 채널을 설정하지 않으면 크론 작업이 실행되지 않을 수 있습니다.

Docker 파일 전송

01

MEDIA: 태그

AI가 메시지에 MEDIA: 태그를 붙이면, 게이트웨이가 해당 파일을 텔레그램으로 전송합니다.

예: MEDIA: /data/report.pdf

경로는 컨테이너 내부 경로입니다.

02

docker_volumes 매핑

Docker 컨테이너에서 파일 공유를 위해 docker_volumes 설정이 필요합니다.

호스트와 컨테이너 간 디렉토리를 마운트하여 파일이 양방향으로 전달됩니다.

이미지: jpg, jpeg, png, gif, webp, bmp, tiff, svg
오디오: mp3, wav, ogg, m4a, flac
비디오: mp4, avi, mov, mkv, webm
