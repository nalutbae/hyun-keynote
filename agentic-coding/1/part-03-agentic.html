강조: "감독관이 되려면, 코드를 읽을 수 있어야 합니다. 그래서 파이썬을 배우는 것입니다."

비교 설명 · 08:00–13:00

양쪽을 하나씩 읽어줍니다. 특히 "수일~수주" vs "수분~수시간"의 차이를 강조합니다.

"전통적 코딩은 문법을 외우고, 에러가 나면 구글링을 하고, 수정하고, 다시 실행하는 과정을 반복합니다. 며칠, 심하면 몇 주가 걸리죠."

"에이전틱 코딩은 '이런 기능이 필요해'라고 AI에게 말하면, AI가 코드를 짜고, 검색도 하고, 에러도 스스로 수정합니다. 몇 분에서 몇 시간이면 끝납니다."

은행 비유: "수기 장부 작성 vs 전산 시스템 입력이라고 생각하시면 됩니다. 결과물은 같아도, 도달하는 속도와 노력이 완전히 다릅니다."

주의: "하지만 빠르다고 해서 검증이 필요 없는 것은 아닙니다. 다음 슬라이드에서 그 이유를 설명합니다."

에이전틱 코딩 · 1회차 · KEYNOTE 1-3-1

감독관은 대본을 읽을 줄 알아야 한다

AI가 다 해주는데 왜 파이썬을 배우나요?코드를 읽을 수 없으면 결과를 검증할 수 없습니다. 보안 문제, 비즈니스 로직 오류, 작은 버그 수정 — 이것을 판단하는 것은 결국 사람입니다.

왜 파이썬을 배우는가 · 13:00–17:00

"가장 많이 받는 질문입니다. AI가 다 해주면 우리가 왜 코딩을 배워야 하냐고요."

비유: "연극 감독을 생각해봅시다. 감독은 직접 연기를 하지 않습니다. 하지만 대본을 읽고, 배우의 연기가 맞는지 판단하려면 대본을 이해할 수 있어야 합니다. 안 그러면 배우가 대사를 틀려도 모르죠."

실제 사례: "AI가 만든 코드에 보안 구멍이 있을 수 있습니다. 대출 심사 로직이 틀릴 수 있습니다. 이것을 코드를 안 읽으면 절대 발견할 수 없습니다."

"그래서 우리가 배우는 것은 '코드를 짜는 능력'이 아니라 '코드를 읽고 검증하는 능력'입니다. 파이썬 기초만으로도 충분합니다."

왜 Hermes Agent인가 · 37:00–39:00

"왜 Hermes Agent를 선택했는지 설명드리겠습니다."

"여러 AI 코딩 도구가 있습니다. Cursor, GitHub Copilot, Windsurf... 하지만 이런 도구들은 개발자를 위해 만들어졌습니다. VS Code를 켜야 하고, 단축키를 알아야 하고, 코딩 문법을 이미 아는 사람을 위한 도구죠."

"Hermes Agent는 다릅니다. 카카오톡이나 슬랙처럼 채팅으로 대화하듯 AI에게 요청합니다. 결과도 채팅창에서 확인합니다. 익숙한 방식이죠."

"그리고 점차 코드를 이해하게 되면, 더 복잡한 요청도 가능합니다. 이게 '학습 경로'입니다. 처음부터 코드를 짜야 하는 것이 아니라, 채팅으로 시작해서 점점 깊어지는 것입니다."

"우리 커리큘럼의 철학과 정확히 맞아떨어집니다."
