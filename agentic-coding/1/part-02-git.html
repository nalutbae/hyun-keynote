Git(로컬)

에이전틱 코딩 · 1회차 · KEYNOTE 1-2-2

SSH 키 설정 3단계

1. 키 생성

ssh-keygen -t ed25519

VS Code 터미널에서 명령어를 실행합니다. 저장 위치와 비밀번호는 엔터로 넘어갑니다.

2. 공개 키 복사

type %USERPROFILE%\.ssh\id_ed25519.pub

터미널에 출력된 내용을 전부 드래그하여 복사합니다. 이게 GitHub에 등록할 열쇠 복사본입니다.

3. GitHub 등록

Settings → SSH keys → New SSH key

GitHub의 Settings → SSH and GPG keys → New SSH key에서 복사한 공개 키를 붙여넣기합니다.

에이전틱 코딩 · 1회차 · KEYNOTE 1-2-4

.gitignore: Git이 무시해야 할 파일 목록

비밀번호 보호

민감 정보는 절대 GitHub에 올리지 않는다

.env 파일에는 API 키, 비밀번호 등이 들어있다. 이 파일은 .gitignore에 반드시 등록해야 한다. 은행 금고 비밀번호를 공개 게시판에 적는 것과 같다.

불필요 파일 제외

실행에 필요 없는 파일은 올리지 않는다

__pycache__/, venv/, .DS_Store 등은 프로젝트 실행에 불필요하며, 올리면 저장소가 지저분해진다.

1-2-4. .gitignore13 / 14
● SPEAKER NOTESSLIDE 13 / 14

.gitignore · 41:00-45:00

  • "파트 1에서 .env 파일이 금고 비밀번호 보관함이라고 했죠? .gitignore는 '이 파일은 GitHub에 올리지 마!'라고 Git에게 미리 알려주는 목록입니다."
  • 비밀번호 보호: "실제로 많은 초보자가 .env 파일을 GitHub에 올려서 API 키가 유출되는 사고를 겪습니다. 특히 은행에서 쓰는 API 키가 유출되면? 상상하기도 싫죠. .gitignore에 .env를 추가하는 것은 보안의 기본입니다."
  • 불필요 파일: "__pycache__/는 파이썬이 자동으로 만드는 임시 파일이고, venv/는 가상환경 폴더입니다. 이들은 프로젝트마다 다시 생성되므로 올릴 필요가 없습니다."

실습: 프로젝트 폴더에 .gitignore 파일을 만들고 .env, __pycache__/, venv/를 한 줄씩 적어보게 합니다.
