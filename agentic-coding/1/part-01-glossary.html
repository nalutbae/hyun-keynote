코딩의 세 기둥: 실행, 디버깅, 변수

실행

Execute / Run

작성한 코드를 컴퓨터가 읽고 수행하는 것. VS Code에서 ▷ 버튼이나 터미널 명령어로 실행한다.

디버깅

Debugging

코드에서 오류를 찾아 고치는 작업. 컴퓨터 안에 날아든 벌레(bug)를 제거했다는 실제 일화에서 유래.

변수

Variable

데이터를 담아두는 상자. 이름 = '김은행'처럼 값을 넣고 필요할 때 꺼내 쓴다.

개발 환경의 세 가지 핵심 도구

IDE

VS Code

코드 작성, 실행, 디버깅을 한 곳에서. 전 세계 개발자의 74%가 사용하는 무료 코드 편집기.

패키지

Library / Package

다른 사람이 미리 만들어둔 코드 묶음. 마트에서 파는 시판용 만두피처럼, 바로 가져다 쓴다.

가상환경

Virtual Env

프로젝트마다 독립된 공간. 이탈리아 요리와 한식을 동시에 하면 냄새가 섞이듯, 가상환경은 각 요리에 독립된 주방을 제공.

Git과 GitHub: 코드의 타임머신과 클라우드

Git

코드의 타임머신

파일이 바뀔 때마다 스냅샷을 찍듯 변경 이력을 기록. 실수해도 언제든 과거로 돌아갈 수 있다. 리누스 토르발스가 2005년에 만듦.

GitHub

코드의 클라우드 저장소

Git으로 관리하는 코드를 인터넷에 올려 백업·공유. 네이버 MYBOX의 개발자 버전. 강사와 코드를 공유하며 첨삭받는 공간.

에이전틱 코딩의 세 가지 핵심 용어

에이전틱 코딩

Agentic Coding

AI 에이전트에게 자연어로 업무를 지시하면, AI가 스스로 판단·검색·실행하여 결과를 내는 개발 방식.

프롬프트

Prompt Engineering

AI에게 최적의 지시를 작성하는 기술. 좋은 프롬프트가 좋은 결과물을 만든다.

하네스

Harness

에이전트가 안전하게 작동하도록 제약과 가이드를 제공하는 프레임워크. 자동차의 차선과 신호등 같은 역할.
