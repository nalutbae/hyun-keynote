첫 번째 모듈은 AI의 발전 과정입니다. AI가 4단계로 진화했다는 점을 먼저 이야기합니다.

"AI라고 하면 ChatGPT를 떠올리시겠지만, 사실 AI의 역사는 훨씬 깁니다. 오늘은 4단계로 나누어서 설명하겠습니다. 각 단계가 어떻게 다른지, 왜 다음 단계로 넘어갔는지를 이해하는 것이 중요합니다."

에이전틱 AI · 1회차 · KEYNOTE 1-1-1

AI는 4단계를 거쳐 진화했습니다

규칙 기반 → 머신러닝 → 생성형 AI → 에이전틱 AI각 단계마다 인간의 개입 방식이 근본적으로 바뀝니다.에이전틱 AI는 인간의 의도를 이해하고 스스로 행동하는 단계입니다.
1-1-1. AI 발전 과정03 / 18
● SPEAKER NOTESSLIDE 03 / 18

핵심 개념 · 03:00–06:00

4단계를 한 줄로 소개합니다. 각 단계의 핵심 차이는 '인간의 개입 정도'입니다.

"규칙 기반은 사람이 모든 규칙을 직접 짜야 했습니다. 머신러닝은 데이터를 주면 패턴을 찾아줍니다. 생성형 AI는 질문에 답을 만들어냅니다. 그리고 에이전틱 AI는 목표만 주면 스스로 계획하고 실행합니다."

강조: "에이전틱 AI의 핵심은 '자율성'입니다. 사람이 구체적인 지시를 하나하나 내리는 게 아니라, 목표만 말하면 AI가 알아서 길을 찾아갑니다."

에이전틱 AI · 1회차 · KEYNOTE 1-1-1

1단계: 규칙 기반 AI (~2000년대 초반)

01

동작 원리

사람이 "A면 B하라"는 규칙을 직접 코딩합니다. IF-THEN 규칙 기반으로, 예외 상황을 모두 사람이 미리 정해야 합니다.

02

활용 형태

신용카드 부정 탐지, 스팸 메일 필터링, 전문가 시스템(의료 진단 보조). 규칙이 복잡해질수록 유지보수가 불가능해집니다.

03

한계

새로운 상황에 대응하지 못합니다. 규칙에 없는 패턴은 처리 불가. "규칙의 저주"라고 부르는 현상이 발생합니다.

04

한계 예시

은행에서 "5만 원 이상 해외 거래는 차단" 규칙 → 출장 중인 고객이 정상 결제도 불가. 예외 처리 규칙이 끝없이 늘어납니다.

1-1-1. AI 발전 과정04 / 18
● SPEAKER NOTESSLIDE 04 / 18

1단계 설명 · 06:00–12:00

규칙 기반 AI는 여러분이 가장 쉽게 이해할 수 있는 형태입니다. 은행 업무와 비슷하게 생각해보세요.

"은행에서 대출 심사 규정을 생각해보세요. '연소득 4천만 원 이상, 신용등급 5등급 이내' 같은 조건을 직접 만들어야 합니다. 규칙이 100개, 1000개가 되면, 어느 규칙이 충돌하는지도 모르게 됩니다. 이게 규칙 기반 AI의 치명적 한계입니다."

스팸 필터 예시: "제목에 '무료'가 있으면 스팸" → 정상 메일도 걸러짐. 규칙 추가 → 또 예외 발생. 끝없는 싸움.

에이전틱 AI · 1회차 · KEYNOTE 1-1-1

2단계: 머신러닝 & 딥러닝 (2010년대)

01

동작 원리

데이터를 주면 AI가 스스로 패턴을 학습합니다. 사람이 규칙을 짜는 게 아니라, 데이터에서 규칙을 찾아냅니다.

02

활용 형태

추천 시스템(넷플릭스, 유튜브), 음성 인식(Siri), 이미지 분류(의료 영상 진단), 이상 탐지(금융 사기 탐지).

03

한계

학습 데이터에 없는 상황은 처리 불가. "블랙박스" 문제로 왜 그런 결론을 냈는지 설명이 어렵습니다.

04

전환점

2012년 ImageNet 대회에서 딥러닝이 압승. 2016년 알파고 vs 이세돌이 대중적 인식의 전환점이 되었습니다.

● SPEAKER NOTESSLIDE 05 / 18

2단계 설명 · 12:00–18:00

머신러닝은 규칙을 사람이 짜지 않아도 된다는 점에서 혁신적이었습니다. 하지만 여전히 인간이 많은 개입을 해야 합니다.

"규칙 기반에서는 사람이 규칙을 짰다면, 머신러닝은 데이터를 주면 패턴을 찾아냅니다. 넷플릭스가 '이 영화를 본 사람들이 좋아할 영화'를 추천하는 게 머신러닝입니다."

한계 강조: "하지만 학습 데이터에 없는 건 모릅니다. 2020년 코로나 같은 전례 없는 상황에서는 머신러닝 모델들이 예측에 실패했습니다. 또 '왜 이 결론을 냈는지' 설명하기 어려운 블랙박스 문제도 있습니다."

에이전틱 AI · 1회차 · KEYNOTE 1-1-1

3단계: 생성형 AI (2022~2024)

01

동작 원리

자연어로 질문하면 AI가 답변, 코드, 이미지 등을 생성합니다. 프롬프트(지시문)가 핵심 입력입니다.

02

활용 형태

챗봇(ChatGPT, 클로드), 코드 생성(Copilot), 문서 요약, 번역, 이미지 생성(미드저니, DALL-E).

03

한계

한 번에 하나의 질문에만 답합니다. 복잡한 업무 흐름은 스스로 처리하지 못합니다. 할루시네이션(환각) 문제도 존재합니다.

04

은행 예시

"이 계약서의 리스크를 분석해줘" → 분석 결과를 주지만, 실제 조치는 인간이 직접 해야 합니다. AI는 조언자일 뿐입니다.

3단계 설명 · 18:00–24:00

생성형 AI는 여러분이 가장 잘 아는 단계입니다. ChatGPT를 써보신 분?

"ChatGPT가 2022년 말에 나왔을 때, 세상이 바뀌었다고들 했죠. 질문만 하면 답을 주니까요. 하지만 한계가 있습니다. 한 번에 하나의 질문에만 답합니다. '이 계약서 리스크 분석해줘'라고 하면 분석은 해주지만, 다음에 뭘 해야 하는지는 알려주지 않습니다."

은행 비유: "생성형 AI는 똑똑한 컨설턴트와 같습니다. 질문하면 답은 잘해주지만, 스스로 업무를 진행하지는 않습니다. 여전히 인간이 각 단계에서 개입해야 합니다."

에이전틱 AI · 1회차 · KEYNOTE 1-1-1

4단계: 에이전틱 AI —
목표만 말하면 알아서 한다

생성형 AI가 조언자라면, 에이전틱 AI는 실행자입니다."이 계약서 리스크 분석하고, 이상 항목은 법무팀에 메일 보내줘"라고 하면,AI가 계획을 세우고, 필요한 도구를 호출하며, 결과를 검증합니다.

4단계 핵심 · 24:00–28:00

에이전틱 AI의 본질을 설명합니다. "조언자 vs 실행자" 비유가 가장 직관적입니다.

"지금까지의 AI는 질문에 답하는 조언자였습니다. 에이전틱 AI는 목표를 주면 스스로 계획을 세우고 실행하는 실행자입니다."

은행 비유: "생성형 AI는 '이 계약서의 리스크가 뭔가요?'라고 물어보면 답을 줍니다. 에이전틱 AI는 '계약서 100개 리스크 분석해줘'라고 하면, 파일을 열고, 분석하고, 이상 항목은 법무팀에 메일까지 보내줍니다."

Cursor — 에이전틱 코딩의 시초

01

개념

VS Code 기반 에디터에 AI를 통합. 코드 작성 중 자동완성, 채팅으로 코드 수정, 멀티 파일 편집을 지원합니다.

02

강점

기존 개발 워크플로우에 자연스럽게 통합. IDE 안에서 벗어나지 않고 AI를 활용할 수 있습니다.

03

한계

에디터 환경에 종속. 터미널 명령, 파일 시스템 조작 등 에디터 밖 작업에는 제한적입니다.

04

의의

"AI가 내 옆에서 코딩을 돕는다"는 개념을 처음으로 제대로 구현한 도구. 에이전틱 코딩의 출발점.

에이전틱 AI · 1회차 · KEYNOTE 1-1-2

OpenClaw — 오픈소스 에이전틱 코딩

01

개념

터미널 기반 오픈소스 코딩 에이전트. 에디터 없이도 파일 읽기, 쓰기, 터미널 명령 실행이 가능합니다.

02

강점

에디터 환경을 벗어나 터미널에서 자유롭게 작업. 오픈소스이므로 커스터마이징과 확장이 자유롭습니다.

03

한계

설정이 복잡하고, 초보자가 쓰기에는 진입 장벽이 높습니다. 커뮤니티 중심이라 안정성이 떨어질 수 있습니다.

04

의의

"에디터를 벗어나 터미널에서 AI가 작동한다"는 가능성을 보여줌. 코딩 에이전트의 범위를 넓힌 도구.

"하지만 설정이 복잡하고, 에러 메시지도 영어로 나오니까 처음 쓰는 분들은 어려울 수 있습니다."

에이전틱 AI · 1회차 · KEYNOTE 1-1-2

Claude Code — 자율적 코딩 에이전트

01

개념

Anthropic의 Claude를 기반으로 한 터미널 코딩 에이전트. 자율적으로 코드를 읽고, 수정하고, 테스트까지 실행합니다.

02

강점

높은 자율성. "이 버그를 고쳐줘"라고 하면 파일을 찾고, 원인을 분석하고, 수정까지 자동으로 진행합니다.

03

한계

코딩에 특화. 코딩 외 업무(이메일, 문서 작성, 데이터 분석 등)에는 사용 불가. Claude 모델에 종속.

04

의의

"자율적 코딩"이라는 개념을 현실로 만든 도구. 하지만 여전히 코딩 영역에만 국한된다는 한계가 있습니다.

Claude Code 설명 · 37:00–40:00

생성형 AI

질문에 답하는 조언자

목표를 달성하는 실행자

  • 도구 호출 가능 (파일, API, 터미널)
  • 장기 기억과 컨텍스트 유지
  • 예: Hermes Agent, Claude Code
