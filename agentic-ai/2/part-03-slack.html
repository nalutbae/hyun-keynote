공개 URL 없이 연결합니다

WebSocket 기반 연결이므로 방화벽 뒤, 랩탑, 사설 서버에서도 작동합니다.인증에는 xoxb- 접두어의 Bot Token과 xapp- 접두어의 App-Level Token이 필요하며,사용자 식별은 Slack Member ID(U01ABC2DEF3 형식)로 이루어집니다.

Step 2: Bot Token Scopes (11개 필수)

01

chat:write

메시지 전송. 봇이 채널과 DM에 메시지를 보낼 수 있는 권한입니다.

02

app_mentions:read

@멘션 이벤트 수신. 채널에서 봇이 멘션되었을 때 감지합니다.

03

channels:history ⚠️

공개 채널 메시지 조회. 가장 많이 누락되는 스코프입니다. 없으면 채널에서 응답 불가.

04

channels:read

공개 채널 정보 조회. 채널 목록과 메타데이터에 접근합니다.

05

groups:history ⚠️

비공개 채널 메시지 조회. 누락 시 비공개 채널에서 응답 불가.

06

im:history

DM 메시지 조회. 1:1 대화 기록에 접근합니다.

07

im:read

DM 채널 정보 조회. DM 목록과 메타데이터에 접근합니다.

08

im:write

DM 채널 열기. 사용자가 처음 DM을 보낼 때 채널을 엽니다.

09

users:read

사용자 정보 조회. Member ID와 사용자 이름을 매핑합니다.

10

files:read ⚠️

파일 조회. 누락 시 이미지·문서 첨부 처리 불가.

11

files:write

파일 업로드. AI가 생성한 파일을 슬랙에 전송합니다.

Step 3: Socket Mode 활성화

01

App-Level Token 생성

Features → Basic Information → App-Level Tokens에서 생성합니다.
토큰 이름: 예) hermes-socket
스코프: connections:write 추가

02

Socket Mode 활성화

Settings → Socket Mode에서 Enable Socket Mode를 켭니다.
앞서 생성한 App-Level Token이 자동으로 연결됩니다.

03

xapp- 토큰 복사

생성된 xapp- 접두어 토큰을 안전한 곳에 복사합니다.
이 토큰이 SLACK_APP_TOKEN 환경변수에 들어갑니다.

04

⚠️ 주의사항

connections:write 스코프가 없으면 Socket Mode 연결이 거부됩니다. 토큰은 절대 공개 저장소에 커밋하지 마세요.

2-3-2. Socket Mode 및 이벤트 설정07 / 18
● SPEAKER NOTESSLIDE 07 / 18

Socket Mode 활성화 · 16:00–20:00

Socket Mode 활성화 과정을 단계별로 설명합니다.

"App-Level Token은 Bot Token과 다릅니다. Bot Token은 xoxb-로 시작하고, App-Level Token은 xapp-로 시작합니다. Socket Mode 연결에는 xapp- 토큰이 필요합니다."

"connections:write 스코프를 반드시 추가해야 합니다. 이게 빠지면 게이트웨이가 슬랙에 연결하지 못합니다."

"Socket Mode를 켜면 슬랙이 WebSocket 연결을 열어주고, Hermes는 이 연결을 통해 실시간으로 메시지를 주고받습니다. 공개 URL이 필요 없는 이유가 바로 여기에 있습니다."

에이전틱 AI · 2회차 · KEYNOTE 2-3-2

Step 4: 이벤트 구독

필수

message.im

DM(다이렉트 메시지) 수신 이벤트. 1:1 대화에서 봇이 메시지를 받으려면 필수입니다.

필수

message.channels

공개 채널 메시지 수신. 봇이 초대된 공개 채널의 메시지를 받습니다.

권장

message.groups

비공개 채널 메시지 수신. 비공개 채널에서도 봇이 응답하려면 필요합니다.

필수

app_mention

@멘션 이벤트 수신. 채널에서 @봇이름으로 멘션되었을 때 감지합니다.

⚠️

#1 설정 이슈

DM은 되는데 채널에서 안 되면 99%가 이벤트 구독 누락입니다. message.channels 또는 message.groups가 빠져 있는지 확인하세요.

💡

설정 위치

Features → Event Subscriptions → Enable Events → Bot Events에서 위 4개 이벤트를 추가합니다.

이벤트 구독 · 20:00–24:00

Step 6: 워크스페이스에 앱 설치

01

앱 설치

Settings → Install App → Install to Workspace를 클릭하고, 권한을 검토한 후 Allow를 클릭합니다.

02

xoxb- 토큰 복사

설치 완료 후 Features → OAuth & Permissions에서 Bot User OAuth Token(xoxb- 접두어)을 복사합니다.

03

⚠️ 재설치 필수

Bot Token Scopes나 이벤트 구독을 변경할 때마다 반드시 앱을 다시 설치해야 합니다. 그렇지 않으면 변경 사항이 반영되지 않습니다.

04

토큰 관리

xoxb- 토큰은 SLACK_BOT_TOKEN 환경변수로 사용됩니다. xapp- 토큰과 함께 안전하게 보관하세요.

Step 8: Hermes 설정

필수

환경변수 설정

SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
SLACK_ALLOWED_USERS=U01ABC2DEF3

세 가지 환경변수를 ~/.hermes/.env 파일에 설정합니다.

선택

홈 채널 설정

SLACK_HOME_CHANNEL=C01XYZ456ABC
SLACK_HOME_CHANNEL_NAME=general

홈 채널은 크론 작업 결과를 전송할 기본 채널입니다.

추천

hermes gateway setup

hermes gateway setup을 실행하면 대화형 마법사가 환경변수 설정을 안내합니다.
직접 .env 파일을 편집하는 것보다 권장합니다.

⚠️

주의사항

SLACK_ALLOWED_USERS에 등록되지 않은 사용자는 봇과 대화할 수 없습니다.
여러 사용자는 쉼표로 구분합니다.
